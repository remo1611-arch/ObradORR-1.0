SwiftRemo SQL v1.0 RC9_MODULOS_LIMPIOS

Fase 3 aplicada sobre RC8.

Objetivo:
- Reducir arquitectura híbrida visible y nombres de módulos versionados sin romper la app.

Cambios:
- Renombrados módulos de entrada públicos a nombres de dominio estables:
  bakery.js, preferments.js, extras.js, culinary.js, history.js, combobox.js y print.js.
- Actualizados imports y scripts de sqlite.html a ?v=100rc9.
- Mantenida la estructura por dominios existente: app.js, db-service.js, storage-service.js, repositories.js, ui, domain y módulos funcionales.
- Eliminados scripts sueltos inline_script_* usados solo como artefacto de validación.
- Renombradas funciones RC8 del render activo a nombres neutros: renderActiveRoute, renderRouteSet, safeRenderRouteStep y currentResolvedRoute.

Decisiones de compatibilidad:
- No se renombran todavía ids HTML ni funciones V6xx porque están acoplados a selectores, claves de localStorage/IndexedDB y flujos legacy activos.
- No se fusionan módulos grandes todavía para evitar una refactorización masiva fuera de fase.

No aplicado todavía:
- Lazy DOM de sqlite.html.
- Paginación SQL real.
- Caché estable/invalidez.
- Rediseño de persistencia.
