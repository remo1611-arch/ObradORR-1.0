SwiftRemo SQL v1.0 RC10_LAZY_DOM

Objetivo
- Reducir peso de sqlite.html y evitar que las vistas completas no abiertas vivan como DOM activo desde el arranque.

Cambios aplicados
- Las pantallas view-* se han movido a <template data-lazy-view="...">.
- app-shell.js materializa cada vista solo cuando se navega a su ruta.
- Las vistas materializadas se mueven a su host de dominio y quedan ocultas/visibles como antes.
- app.js vuelve a enlazar controles dinámicos cuando una vista se materializa.
- Se conservan los módulos iniciales; la descarga modular bajo demanda queda para una fase posterior si se mide como necesaria.
- Se añaden marcas boot lazy-dom:view-instantiated y lazy-dom:controls-bind.

Límites conscientes
- No se ha vaciado el HTML hasta un shell puro para no romper selectores ni editores complejos.
- Los nombres legacy internos V6xx se conservan.
- No se aplica todavía paginación SQL real ni caché de catálogos.
