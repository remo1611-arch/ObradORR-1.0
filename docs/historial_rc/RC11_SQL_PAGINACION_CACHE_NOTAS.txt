RC12_HOTFIX_CATALOGO_PAGINADO

Objetivo de fase:
Hacer que las listas principales escalen mejor evitando traer todos los registros para paginar en JavaScript.

Cambios aplicados:
1. Repository incorpora pagedQuery(), ingredientsPage() y unifiedElaborationsPage().
2. Ingredientes y elaboraciones usan COUNT + LIMIT + OFFSET.
3. Las funciones legacy ingredients() y unifiedElaborations() se conservan por compatibilidad, pero delegan en las versiones paginadas con pageSize='all'.
4. El editor avanzado de ingredientes incorpora selector 10/50/100/Todos y paginadores superior/inferior.
5. El editor avanzado de elaboraciones deja de hacer slice JS sobre todas las elaboraciones para la página visible.
6. Catálogos técnicos y auditoría quedan cacheados en Repository.
7. La caché se invalida al guardar/desactivar ingredientes.

Límites deliberados:
- La biblioteca técnica general sigue aplicando parte de su auditoría/calidad en memoria, porque qualityReview() depende de reglas JS y de líneas/procesos de fichas. Migrarlo a SQL completo exige una fase propia.
- No se ha cambiado el modelo de datos ni la persistencia.
- No se ha reintroducido lazy DOM por templates tras los fallos de RC10/RC10a.

Prueba esperada:
Abrir Archivo técnico > Elaboraciones e Ingredientes; cambiar page size; pasar página; buscar; volver a 50; comprobar que la consola no emite errores críticos.
